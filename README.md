## Python Functions & Functional Programming

Today, I focused on understanding Python functions and explored the fundamentals of functional programming.

### Topics Covered

- Learned how Python functions work and how arguments are handled at runtime.
- Practiced using **default arguments** and understood why they should be placed at the end of the parameter list.
- Explored **type hints** to improve code readability.
- Worked with **variable-length positional arguments (`*args`)**.
- Learned **variable-length keyword arguments (`**kwargs`)** and iterated through dictionaries using `.items()`.
- Built a simple **order processing system** using `*args` and `**kwargs`.
- Understood the difference between **local** and **global** variables, including the use of `globals()`.
- Implemented **higher-order functions** by passing functions as arguments.
- Applied different discount strategies using function references.
- Learned and used **lambda functions** for concise anonymous functions.
- Practiced Python's functional programming utilities:
  - `map()`
  - `filter()`
  - `reduce()` (concept)
- Used built-in functions like `int` as callback functions.

### Key Takeaways

- Python functions are **first-class objects**, meaning they can be passed, returned, and assigned like any other object.
- `*args` collects extra positional arguments into a tuple.
- `**kwargs` collects extra keyword arguments into a dictionary.
- Lambda functions simplify small, single-expression operations.
- Functional programming tools like `map()` and `filter()` make code more concise and expressive.
